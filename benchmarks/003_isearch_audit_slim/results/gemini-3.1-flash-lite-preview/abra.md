# Концептуальный Протокол: Аудит `isearch`

## 1. Топология задачи
- **Мета-тип:** Инфраструктурный аудит векторной поисковой системы.
- **Ключевой конфликт:** Желание обеспечить скорость (упрощенная инфраструктура) вступает в противоречие с целостностью данных (недетерминированные ID, гонки состояний).

## 2. Инварианты
1. **Идемпотентность индекса:** Повторный запуск индексации одного и того же контента должен приводить к обновлению существующих записей, а не дублированию.
2. **Производительность фильтрации:** Фильтры поиска должны быть поддержаны индексами в БД.
3. **Оркестрация:** Управление жизненным циклом должно быть отвязано от текущей сессии shell.

## 3. Точка опоры
- **Focus:** Замена `hash()` на детерминированный `hashlib` для ID и создание индексов для всех полей фильтрации (`file_type`).

## 4. Векторы энтропии
- **Коллизии:** Использование `hash()` в Python приводит к «раздуванию» БД при каждом реиндексе.
- **Full Scan:** Поиск по `scope` превращается в O(N) операцию, деградирующую систему при росте объема.

## 5. Аудит (Findings)

### Критические (Critical)
1. **[Infrastructure] Недетерминированные ID в Qdrant (`vector_store.py`):**
   - *Root Cause:* `ids = [abs(hash(p.get('source_file', '') + p.get('text', ''))) for p in payloads]`.
   - *Impact:* `hash()` меняется между запусками. Старые данные не удаляются, новые дублируются. База будет расти до OOM.
2. **[Architecture] Отсутствие индекса для фильтрации (`vector_store.py`):**
   - *Root Cause:* Фильтр по `file_type` (в `main.py`) не имеет соответствующего `create_payload_index` в `vector_store.py`.
   - *Impact:* Запросы `scope=code/docs` выполняются как полнотекстовый перебор всех векторов в коллекции.

### Высокие (High)
3. **[Design] Нестабильная демонизация (`manage.py`):**
   - *Root Cause:* Использование `nohup ... &` и ручное управление PID-файлом.
   - *Impact:* "Зомби"-процессы, невозможность корректного перезапуска при падении, отсутствие связи с systemd/docker-оркестрацией.
4. **[Product] Ошибка группировки/ранжирования (`main.py`):**
   - *Root Cause:* Логика бустинга (boost_factor) срабатывает только если `scope` не задан. Если пользователь явно запросил `scope=code`, бустинг не применяется, что может ухудшить релевантность.

### Средние (Medium)
5. **[Code] Ресурсоемкость:** Использование `KMeans` на CPU при больших объемах векторов в `/projects/{project_name}/clusters` без пагинации/лимитов.

## 6. Метрика истины
- Реиндексация проекта N раз подряд должна возвращать константный `points_count` в Qdrant.
- Запрос с `scope=docs` должен быть мгновенным (использование индексов).

## 7. Эвристики
- **Рекомендация:** Использовать `hashlib.sha256().hexdigest()` с последующим `int(..., 16) % (2**64)` для генерации ID.
- **Рекомендация:** Добавить `self.client.create_payload_index(..., "file_type", ...)` в `VectorStore.recreate_collection`.
