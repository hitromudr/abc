# Benchmark 004: Bug Fix — Nondeterministic Hash IDs

## Задача

В файле `src/vector_store.py` метод `upsert()` генерирует point ID через `abs(hash(...))`.

`hash()` в Python использует рандомизированное зерно (PYTHONHASHSEED) — ID меняются между перезапусками процесса. Это приводит к:
- Дублированию точек при reindex (insert вместо update)
- Экспоненциальному росту коллекции
- Несогласованности поиска

## Требования к фиксу

1. Заменить `hash()` на детерминистичную хеш-функцию (e.g. SHA-256)
2. ID должен быть **положительным int64** (ограничение Qdrant)
3. Результат стабилен между перезапусками процесса
4. Минимальный патч — не трогать логику батчинга, search и другие методы

## Падающий тест

`tests/unit/test_id_determinism.py` — проверяет стабильность ID при одинаковом input.
