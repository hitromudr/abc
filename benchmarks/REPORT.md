## 6. Cadabra vs GSD в интерактивном режиме (bench 005)

### Что такое GSD-агент

GSD (Get Shit Done) — монолитный агент, который одновременно и Архитектор, и Исполнитель. Стандартный паттерн Claude Code / Cursor: «ты сеньор, вот задача, пиши код, гоняй тесты, чини ошибки, пока не позеленеет». Доступ к файлам, shell, тестам.

### Условия

Та же задача bench 005 (консолидация 3× gitignore). Модель: дешёвая/быстрая (Gemini Flash / Qwen 9B уровень). Два режима:
- **GSD + Tools** — монолитный агент с доступом к файлам, shell, тестам. Без ограничений.
- **Cadabra + Tools** — стейт-машинный агент с EXECUTION_STATE.md от abra. DAG, Kill Box, retry budget.

### Результаты

| Метрика | GSD + Tools | Cadabra + Tools | Δ |
|---------|:-----------:|:---------------:|:-:|
| tests_pass | ❌ (0%) | ✅ (100%) | 0% → 100% |
| API preserved | ❌ ломает | ✅ сохранён | — |
| diff size (LOC) | ~350–400 | ~80 | -78% |
| cost | ~$0.30 | ~$0.08 | -73% |
| scope creep | да | нет (Kill Box) | — |

### 4 механизма провала GSD на Complicated-задачах

**1. Context Bloat (вымывание контекста).** GSD-монолит — одновременно Архитектор и Исполнитель. После 3-й ошибки окно контекста забито стэктрейсами pytest. Дешёвые модели теряют исходную архитектурную задачу и переходят в режим паники — хакают тесты или сносят соседние методы. Cadabra опирается на EXECUTION_STATE.md: архитектура и ограничения — во внешнем физическом файле, а не в голове агента.

**2. Мутация публичного API (Ось 2: Топология).** Для GSD главное — зелёный свет в консоли. Если для рефакторинга удобнее изменить сигнатуру `build_graph()` (что сломает импорты в другом репозитории), GSD сделает это. Cadabra имеет хардкодный блок MUST_NOT_DO — если контракт запрещает менять публичный API, это физический закон. При невозможности шага без нарушения Kill Box агент останавливается (`blocked` + ERROR_LOG), а не ломает проект.

**3. Экономика отказов (Fail Fast vs Burn Budget).** GSD склонен к зомби-циклам: $1.50 и 15 попыток, бегая по кругу между синтаксическими ошибками до max_turns. Cadabra: локальная верификация (`python -m py_compile`) на каждом шаге. Если файл не компилируется — агент не дойдёт до интеграционных тестов.

**4. Разделение ролей по стоимости.** GSD платит за каждую итерацию по цене одной модели — одна и та же модель думает, пишет, дебажит. Это `O(n × expensive)`. Cadabra: один вызов дорогой модели (abra, архитектор) + N итераций дешёвой (cadabra, исполнитель) = `O(1 × expensive + n × cheap)`.

### Экономика

| Компонент | GSD | abra + cadabra |
|-----------|:---:|:--------------:|
| Архитектура | встроена в каждый turn | 1× дорогая модель ($0.02) |
| Исполнение | N turns × та же модель ($0.30) | N turns × дешёвая модель ($0.08) |
| Kill Box | нет | MUST_NOT_DO в контракте |
| Retry budget | max_turns (бесконечный) | 3 попытки на шаг |
| **Итого** | **$0.30, 0% success** | **$0.10, 100% success** |

## 7. Выводы

### Главный результат

**abracadabra — cost router для агентных задач.** Не "улучшаем мышление LLM", а "делаем Complicated-задачи экономически возможными для дешёвых моделей".

Философия GSD: одна модель думает + пишет + дебажит. `O(n × expensive)`.
Философия cadabra: дорогая модель один раз (архитектурный план) → дешёвая модель итерирует по DAG. `O(1 × expensive + n × cheap)`.

### Что работает
1. **Cadabra + tools побеждает GSD на Complicated-задачах** — 100% success vs 0%, в 3× дешевле. Ключевой результат проекта
2. **Safe delegation** — EXECUTION_STATE.md (DAG + Kill Box + retry budget) превращает Gemini Flash / Qwen 9B в надёжных исполнителей для задач, которые те же модели в GSD-режиме проваливают
3. **Abra помогает слабым моделям на audit** — CoT-рельсы от фреймворка
4. **Slim KB (33KB) оптимален** — Full KB (75KB) вредит флагманам (Lost in the Middle)
5. **GSD оптимален для Clear-задач** — bug fix в one-shot за $0.04–0.06 без фреймворка

### Что не работает
1. **Abra на Clear-задачах** — overhead +634% cost, 0% прироста. GSD оптимален
2. **Cadabra в API (без tools)** — без файловой системы и retry loop бесполезен
3. **GSD на Complicated-задачах** — Context Bloat, мутация API, зомби-циклы, Agentic Drift

### Cynefin-маппинг (эмпирически подтверждён)

| Домен | Класс задачи | Оптимальный подход | Почему |
|-------|-------------|-------------------|--------|
| Clear | Bug fix | GSD (vanilla agent) | Задача конкретная, бинарный критерий |
| Complicated | Refactor, миграция | **abra → cadabra** | GSD теряет контекст, ломает API, зомби-циклы |
| Complex | Audit, архитектура | abra ≈ GSD (паритет) | Abra помогает слабым моделям |
