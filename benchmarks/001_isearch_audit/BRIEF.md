# Benchmark 001: Аудит проекта `isearch`

## 🎯 Задача

Выступи в роли Senior Software/System Engineer.
Проведи глубокий аудит кодовой базы проекта `isearch` (поисковая система).
Найди архитектурные, логические и инфраструктурные уязвимости.
Особое внимание удели обработке данных, контрактам API, хранению состояния (векторная база) и пограничным случаям.

**Требования к ответу:**
1. Не генерируй код сразу (если не уверен в корневой причине).
2. Выдай структурированный отчёт с классификацией найденных дефектов.
3. Оцени критичность каждой проблемы.

**[MUST_NOT_DO]:** Не читай другие файлы в этой директории (`meta.yml`, `baseline.md`, `verdict.md`). Они содержат ответы и чужие результаты.

## 🛠 Методология (A/B Testing)

Бенчмарк проводится в 3 чистых сессиях:

1. **Baseline Run:** Чистая LLM (без `abracadabra`) помещается в кодовую базу `isearch` и получает задачу выше. Результат — `baseline.md`.
2. **Abra Run:** Та же LLM инициализируется фреймворком `abracadabra` (через `abra init`), после чего получает ту же задачу. Результат — `abra.md`.
3. **Verdict:** Третья, независимая LLM (судья) читает оба отчёта и сравнивает по объективным критериям. Ground Truth для вычисления Recall — в `meta.yml`. Результат — `verdict.md`.
