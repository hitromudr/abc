# Концептуальный Протокол: Интеграция Эволюционного Ядра (Virus) в Agentic OS (AutoWarp)

## 1. Топология задачи (Ontology)
- **Мета-тип:** Архитектурный симбиоз детерминированной управляющей среды и стохастического генеративного ядра.
- **Ключевой конфликт (Диалектика):** Линейная хрупкость `AutoWarp` (водопадная модель агентов, падающая при сложных галлюцинациях) *против* Экспоненциальной стоимости `Virus` (эволюционный брутфорс сжигает бюджет).

## 2. Инварианты (Invariants)
1. **Закон Черного Ящика:** `AutoWarp` (Оркестратор) не имеет права управлять внутренними мутациями `Virus`. Он поставляет только контекст (Вход) и Фитнес-функцию (Критерий успеха).
2. **Абсолютная Инкапсуляция:** Эволюционный цикл `Virus` обязан выполняться строго внутри эфемерного `git worktree`, созданного `AutoWarp` (`make init-task`). Выход за пределы task-директории фатален.
3. **Объединенный Метаболизм:** Финансовый Circuit Breaker (лимит токенов) из `Virus` становится подчиненным глобальному бюджету задачи в `task_manager.py` (AutoWarp).

## 3. Точка опоры (Leverage Point)
- **Фокус (TOC):** Замена узкого горлышка — линейного узла `Coder` в `AutoWarp` на вызов эволюционного движка `engine.py` из проекта `Virus`. Переход от парадигмы "Напиши с 3 попыток" к парадигме "Вырасти выживающий код через N поколений".
- **Игнорируемый шум:** Попытки объединить две кодовые базы в единый Python-монолит. Проекты должны оставаться изолированными микросервисами (или CLI-тулзами), общающимися через стандартные потоки (stdin/stdout) или JSON RPC.

## 4. Векторы энтропии (Degradation Paths)
- **Анти-паттерн (Конфликт Апоптозов):** `AutoWarp` имеет свой скрипт очистки (`finalize-task`), а `Virus` — свой (удаление неудачных `plasmids/`). При наложении они могут вызвать состояние гонки (Race Condition) за удаление файлов.
- **Когнитивное искажение (Over-engineering):** Создание сложного API для общения двух систем там, где достаточно передачи exit-кода (0 - Золотой штамм найден, 1 - Бюджет сожжен).

## 5. Алгоритм стабилизации (The Protocol)

- **Фаза 0: Изоляция (Подготовка Контракта):**
  Превратить `virus/engine.py` в CLI-утилиту, принимающую аргументы: `--workdir <путь_к_git_worktree>` `--fitness-cmd <команда_проверки>` `--max-tokens <лимит>`.
- **Фаза 1: Оцифровка базиса (Модификация AutoWarp):**
  В `autowarp/task_manager.py` удалить логику линейного `Coder -> Reviewer`. Вместо этого `Overseer` формирует задачу и делегирует её: запускает подпроцесс `Virus` внутри директории задачи. В качестве `--fitness-cmd` передается вызов тестов AutoWarp (`pytest`).
- **Фаза 2: Применение рычага (Эволюция внутри ОС):**
  `Virus` берет управление. Генерирует мутантов, прогоняет их через `--fitness-cmd`. Тесты `AutoWarp` выступают в роли законов физики для организмов `Virus`. Некомпилируемые штаммы отбраковываются встроенным апоптозом `Virus`.
- **Фаза 3: Интеграция и Стабилизация (Возврат управления):**
  Когда `Virus` достигает `TARGET_FITNESS_SCORE` (тесты зеленые), он делает `git commit` в текущем worktree с меткой `[GOLDEN STRAIN]` и завершается с кодом `0`. `AutoWarp` перехватывает управление, выполняет `make finalize-task`, сливает ветку в `develop` и уничтожает worktree.

## 6. Метрика истины (Truth Criteria)
- **Состояние успеха (Телос):** Успешное прохождение End-to-End теста: `AutoWarp` ставит сложную архитектурную задачу, которая ранее приводила к циклу ошибок. Интегрированный `Virus` решает ее за 5-10 поколений мутаций, и готовый код автоматически мержится в `develop`.
- **Ложный сигнал успеха:** `Virus` генерирует код, проходящий тесты, но оставляющий мусорные файлы (утечка песочницы) или выходящий за лимиты API-бюджета. 

## 7. Предложения, замечания, комментарии (Proposals & Remarks)
- **Открытые вопросы:** Способен ли `Virus` корректно обрабатывать многофайловые мутации в контексте сложных проектов `AutoWarp`, если изначально он тестировался на атомарных скриптах? Потребуется расширение контекстного окна при формировании генома.
- **Тактические предложения (Быстрый Win):** Начать интеграцию без изменения исходников `Virus`. Использовать Bash-обертку, которая копирует `engine.py` в worktree `AutoWarp` и запускает его, собирая артефакты после завершения.
- **Комментарии оператора:** Данный симбиоз переводит `AutoWarp` из класса "Автоматизаторов" в класс "Эволюционных Синтезаторов". Это качественный переход, требующий строжайшего мониторинга стоимости выполнения (Cost of Compute) на каждый таск.
