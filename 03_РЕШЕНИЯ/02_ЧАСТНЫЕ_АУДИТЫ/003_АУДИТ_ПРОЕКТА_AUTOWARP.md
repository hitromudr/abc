# Концептуальный Протокол: Аудит и Патч архитектуры `../autowarp`

**Источник движения:** Изучение архитектуры `AutoWarp` (Agentic Bio-OS) и пайплайнов `task_manager.py`.
**Прогон через Мета-Конвейер:** Успешно (Атака по 8 осям Октагона).

## 1. Топология задачи (Ontology)
- **Мета-тип:** Оркестрация мульти-агентной системы для генерации и модификации кода в локальной файловой системе.
- **Ключевой конфликт (Диалектика):** Автономность агентов (им нужен доступ к файлам для написания кода) *против* Выживания среды (агенты могут уничтожить репозиторий галлюцинациями).

## 2. Инварианты (Invariants)
1. **Закон физической изоляции "Снов":** Тесты и симуляции агента (его "размышления") никогда не должны иметь доступа к реальному диску. Сон не должен оставлять физических следов.
2. **Закон стоимости Ветвления:** Каждое создание физического `git worktree` для задачи — это не бесплатная операция. Она облагается налогом на диск (node_modules, venv), который должен быть выплачен при закрытии задачи.

## 3. Точка опоры и Векторы Энтропии (Атака Октагоном)
Выявлено 3 критических дедлока, ведущих к деградации ОС:

*   **Дыра 1: Пробой Иммунитета (Дедлок `MagicMock`).** 
    В корне репозитория лежат десятки файлов с именами `<MagicMock name='AgentStateBroadcaster().project_root...'>`. Это значит, что ваш тестовый фреймворк (`pytest`) или сам Агент при попытке мокирования файловых путей **физически записал файлы на диск**. Песочница пробита. Если Агент во время тестов "сойдет с ума" и вызовет `os.system("rm -rf .")`, он сотрет проект.
*   **Дыра 2: Рак Диска (Метаболизм vs Топология).** 
    Использование `git worktree` для изоляции задач (`make init-task`) — отличная идея. Но каждый worktree тянет за собой свои зависимости (если проект содержит NPM/Python пакеты). Без жестокого Garbage Collector (Сборщика мусора) в команде `make finalize-task`, диск будет уничтожен дубликатами сред.
*   **Дыра 3: Линейный Водопад (Слепота Кинематики).**
    Пайплайн `Architect -> Coder -> Reviewer` слишком линеен для LLM. Если Coder пишет некомпилируемый код, Overseer пытается его "заставить" переписать. Возникает бесконечный цикл сжигания токенов (Infinite Loop).

## 4. План исправления (The Protocol)

Для лечения системы необходимо внедрить 3 патча:

- **Шаг 1: Аппаратная виртуализация тестов (Патч Иммунитета)**
  Категорически запретить использование стандартного `unittest.mock` для файловых операций в тестах агентов. Внедрить библиотеку `pyfakefs` или использовать фикстуру `tmp_path` из pytest для абсолютной виртуализации файловой системы. Агент должен "галлюцинировать" код только в In-Memory диске.
- **Шаг 2: Агрессивный Апоптоз Задач (Патч Метаболизма)**
  Команда `make finalize-task` должна быть переписана. После мержа в `develop`, скрипт обязан делать: `git worktree remove --force <task_dir>` и `git branch -D <task_branch>`. Никакие "отработанные" ветки не должны оставаться в директории `.worktrees/`.
- **Шаг 3: Внедрение "Kill Switch" в Overseer (Патч Кинематики)**
  Если Coder проваливает тесты 3 раза подряд, Overseer должен лишаться права перезапускать его. Overseer обязан нажать *Kill Switch* (выбросить Exception), откатить `git reset --hard` до начала задачи и вернуть управление человеку-оператору. Это спасет бюджет токенов.

## 5. Чек-лист приемки (Definition of Done)

Инженерная задача по стабилизации `AutoWarp` будет решена при закрытии этих пунктов:

- [ ] **Тест Иммунитета:** Выполнен `make clean` (удалены файлы MagicMock). При полном прогоне `make test` в корне проекта больше не создаются файлы-мутанты.
- [ ] **Тест Апоптоза:** При выполнении `make finalize-task ID=123`, директория `.worktrees/123/` физически исчезает с диска, освобождая мегабайты памяти.
- [ ] **Тест Kill Switch:** В Агента намеренно вбрасывается невыполнимая задача (например, "Реши проблему остановки"). Через 3 итерации Overseer принудительно обрывает процесс, не позволяя сжечь больше 50 000 токенов, и откатывает ветку.
