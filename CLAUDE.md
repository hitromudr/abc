# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это

ABC — набор Markdown-документов, описывающих когнитивную архитектуру (Мета-Конвейер) для обработки сложных задач через AI-промпты. Нет кода, сборки, тестов или линтера. Git используется только для версионирования документации.

Язык — русский. Технические термины на языке оригинала.

## Структура

- `.rules` — системный промпт оператора (роль, правила, workflow)
- `00_ИНИЦИАЛИЗАЦИЯ/` — манифест (философия проекта), Двойная Спираль (аксиома эволюции)
- `01_БАЗА_ЗНАНИЙ/` — 6 документов: эпистемологические фильтры, когнитивный ландшафт, октагон выживания, матрица конфликтов, векторы мышления, органические системы
- `02_ИНСТРУМЕНТЫ/` — алгоритм разбора задачи (UTO v5), шаблон итогового протокола, автономный пайплайн для web-использования, сессионный анализатор, универсальный оптимизатор, скрипт синхронизации контекста
- `03_РЕШЕНИЯ/` — готовые протоколы (нумерация `001, 002...` независимая в каждой поддиректории):
  - `01_ОБЩИЕ_МЕТА_ПРОТОКОЛЫ/`
  - `02_КРОСС_ДОМЕННЫЕ_ИНВАРИАНТЫ/` (6 доменов)
  - `03_ЧАСТНЫЕ_АУДИТЫ/`


## Заметки по совместимости

- `.rules` подхватывается автоматически только **Zed**. Для Cursor нужен `.cursorrules` (копия `.rules`).
- `abc` — не зарегистрированная slash-команда редактора, а текстовый ключ в промпте: AI распознаёт его из `.rules` и запускает pipeline. Работает только если `.rules` загружен в контекст.
- Автономный пайплайн (`02_ИНСТРУМЕНТЫ/03_АВТОНОМНЫЙ_ПАЙПЛАЙН.md`) — самодостаточная сжатая версия pipeline для web-чатов (ChatGPT/Claude). Шаги пронумерованы иначе (0–4), чем в каноническом `01_АЛГОРИТМ_РАЗБОРА_ЗАДАЧИ.md` (Фазы 0–6).

## Правила из .rules (фактические)

1. **NO YAPPING** — запрещены преамбулы, извинения, клише. Только сигнал.
2. **THOUGHT PROCESS** — внутренний монолог в `<thought_process>` перед любым финальным ответом.
3. **THE DOUBLE HELIX** — каждая задача обязана двигать Product (решение) и Process (улучшение системы) одновременно.
4. **ПРОТОКОЛ ВОССТАНОВЛЕНИЯ КОНТЕКСТА** — при инициализации запускается `bash 02_ИНСТРУМЕНТЫ/06_СИНХРОНИЗАЦИЯ_КОНТЕКСТА.sh`, затем параллельное чтение всех файлов. Zed обрезает `read_file` на 16KB, Claude Code — на 2000 строк.
5. **EMPIRICAL GROUNDING** — без метрик/логов задача отклоняется.
6. **SEMANTIC GROUNDING** — абстракция обязана разворачиваться в физический факт в следующем же предложении.

## Формат протоколов

Выходной документ по шаблону `02_ИНСТРУМЕНТЫ/02_ШАБЛОН_ИТОГОВОГО_ПРОТОКОЛА.md`: Топология, Инварианты, Точка опоры, Векторы энтропии, Алгоритм стабилизации, Метрика истины, Предложения/замечания.
