# Мета-Конвейер: UTO Пайплайн (Execution Checklist)

## Фаза 0. Эмпирическое Заземление
- [ ] **Observe:** Отклонить интерпретации. Зафиксировать физически измеримые метрики инцидента.
- [ ] **Verify State:** Верифицировать существование проблемы в логах/ФС.

## Фаза 0.5. Kill Switch (ROI)
- [ ] **Cost of Inaction:** Вычислить момент фатального убытка при бездействии.
- [ ] **Резолюция:** Если `Стоимость Решения > Стоимость Бездействия` — остановить выполнение (Одобренная деградация).

## Фаза 1. Выбор Вектора Тяги
- [ ] **Cynefin:** Определить домен (Complicated, Complex, Chaotic). Если `Complex` — подключить `06_ОРГАНИЧЕСКИЕ_СИСТЕМЫ.md` вместо TOC.
- [ ] **Gearbox 10V:** ЗАДЕКЛАРИРУЙ 1 основной вектор по `05_ВЕКТОРЫ_МЫШЛЕНИЯ.md`.
- [ ] **Lateral Injection:** Выбери 1 контр-вектор для слома когнитивной колеи.

## Фаза 2. Фильтр Уровня 0 (Физика)
- [ ] **Диалектика:** Зафиксировать базовое неразрешимое противоречие (Теза vs Антитеза).
- [ ] **Теория Игр:** Найти равновесие Нэша, удерживающее проблему (почему агентам выгодно не решать ее).
- [ ] **Кибернетика (Закон Эшби):** Локализовать оборванную петлю обратной связи или дефицит сложности управляющей системы.

## Фаза 3. Фильтр Уровня 1 (Оптимизация)
- [ ] **TOC:** Найти единственный ограничитель (Bottleneck) пропускной способности.
- [ ] **Системное мышление:** Выявить балансирующие петли (сопротивление среды).
- [ ] **OODA:** Определить скорость обновления картины мира решателей относительно среды.

## Фаза 4. Фильтр Уровня 2 (Стратегия)
- [ ] **Даосизм:** Описать путь внедрения без административного насилия.
- [ ] **Макиавелли:** Локализовать политическое трение (чьи ресурсы задеты).
- [ ] **Сунь Цзы:** Описать асимметричное решение (как изменить правила, чтобы проблема исчезла).
- [ ] **Стоицизм:** Вычеркнуть из плана зоны вне 100% контроля оператора.

## Фаза 5. Синтез и Атака (Red Teaming)
- [ ] **Scratchpad:** Собрать черновик по `02_ШАБЛОН_ИТОГОВОГО_ПРОТОКОЛА.md`.
- [ ] **Attack:** Атаковать логические дыры. Проверить на соответствие Инвариантам Уровня 0.
- [ ] **Converge:** Отбросить мусор (No Yapping).

## Фаза 6. Архитектурная Коррекция
- [ ] **Проектирование Патча:** На базе TOC сформировать конкретные изменения в архитектуре (код/инфраструктура).
- [ ] **Интеграция:** Записать патч в блок `7. Предложения, замечания, комментарии`.
