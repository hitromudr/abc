# abracadabra

[![MIT License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)

**Когнитивный экзоскелет для LLM — жесткая архитектура вместо промпт-инженерии.**

LLM без ограничений — это океан энтропии: галлюцинации, context rot, слепая угодливость. abracadabra накладывает архитектурные печати — систему когнитивных фильтров, кибернетических ритуалов и матриц выживания, — заставляя модель работать по законам вашей физики.

## Два агента, один ритуал

abracadabra — двухфазный коммит (Two-Phase Commit), воплощающий закон Двойной Спирали (Process & Product):

- **abra** (Архитектор) создаёт **Процесс**. Блокирует инстинкт немедленной генерации кода и прогоняет задачу через 8 философских фильтров и 8 инженерных осей Октагона. Если задача — иллюзия, `abra` её уничтожит. Если реальна — выкует архитектурный контракт.
- **cadabra** (Исполнитель) создаёт **Продукт**. Слепой голем-прораб: берёт контракт от `abra` и методично исполняет шаг за шагом. При сбое — 3 локальные попытки починки внутри разрешённого Scope, затем остановка и возврат управления Архитектору.

```
Оператор → abra [задача] → Концептуальный Протокол → Approval Gate
                                                         ↓
                             Оператор утверждает → abra генерирует EXECUTION_STATE.md
                                                         ↓
                             cadabra [путь к EXECUTION_STATE.md] → DAG → верификация → done/blocked
```

## Quick Start

### Только анализ (любой web-чат)

1. Скачайте [`03_АВТОНОМНЫЙ_ПАЙПЛАЙН.md`](abra/docs/02_ИНСТРУМЕНТЫ/03_АВТОНОМНЫЙ_ПАЙПЛАЙН.md)
2. Прикрепите в чат ChatGPT / Claude / Gemini
3. Напишите задачу — модель выполнит полный цикл анализа

Это только **abra** (анализ и протокол). Для полного цикла (анализ + исполнение) нужна AI-IDE с доступом к файловой системе.

### Полный цикл (AI-IDE)

Клонируйте репо:

```bash
git clone https://github.com/hitromudr/abc.git abracadabra && cd abracadabra
```

| IDE | Настройка | Запуск |
|---|---|---|
| **Zed** | Автоматически через `.rules` | `abra [задача]` в чате |
| **Cursor** | Автоматически через `.cursorrules` | `abra [задача]` в чате |
| **Claude Code** | Автоматически через `CLAUDE.md` | `abra [задача]` в чате |

После получения и утверждения протокола:

```
cadabra ./abra/docs/03_РЕШЕНИЯ/.../EXECUTION_STATE.md
```

cadabra прочитает контракт и начнёт исполнение DAG.

## До / После

**Задача:** *"У нас микросервисы тормозят, что делать?"*

**Без abracadabra** (типичный ответ LLM):
> "Рекомендую оптимизировать запросы к базе данных, внедрить кеширование Redis, рассмотреть переход на gRPC, использовать connection pooling..."

Шаблонный список без диагностики. Не известно, что именно тормозит.

**С abracadabra** (после прогона через конвейер):
> **Задача отклонена (Фаза 0).** Отсутствуют метрики: p99 latency, RPS, flamegraph, трейсы. "Тормозят" — это языковая иллюзия (Фильтр Витгенштейна). Предоставьте: 1) трейс конкретного медленного запроса, 2) график latency за последнюю неделю, 3) текущий CPU/RAM на узлах.

abra заставил модель *отказаться решать несуществующую проблему* и потребовать физические данные.

## Когда НЕ использовать

abracadabra — экзоскелет, а не скафандр. Он для задач, требующих декомпозиции (Complicated/Complex по Cynefin). Не запускайте, если:

- **Задача простая** — известное решение, не нужно 6 фаз
- **Задача творческая** — нарратив, дизайн, брейншторм (формализация убивает нелинейное мышление)
- **Нужна скорость** — ответ за 10 секунд, а не архитектурный документ
- **Нужен разговор** — обсуждение, размышление вслух

## Ключевые механизмы

**8 Абсолютных Фильтров** — пре-чеклист невалидности задачи:

| # | Фильтр | Вопрос |
|---|---|---|
| 1 | Витгенштейн (Язык) | Реальный конфликт или языковая иллюзия? |
| 2 | Декарт (Знание) | В чём я математически уверен? |
| 3 | Сенека (Действие) | Это в зоне контроля? |
| 4 | Аристотель (Цель) | Какова истинная цель (Телос) системы? |
| 5 | Джеймс (Результат) | Решение изменит физическую реальность завтра? |
| 6 | Хаммурапи (Причинность) | Какой базовый контракт нарушен? |
| 7 | Сунь Цзы (Время) | Сейчас ли момент действовать? |
| 8 | Платон (Масштаб) | Я смотрю на правильный уровень? |

**Инженерный Октагон** — 8 осей проверки любого решения:
Топология → Метаболизм → Кинематика → Наследственность → Иммунитет → Гомеостаз → Мутагенез → Телос

**Двойная Спираль** — каждая задача обязана двигать Product (решение) и Process (улучшение системы) одновременно. Одноразовые фиксы запрещены.

**Кросс-доменные инварианты** — фундаментальные законы из 6 областей (термодинамика, кибернетика, биология, теория игр, когнитивистика, семиотика), заземлённые в инженерные паттерны.
