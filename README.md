# ABC: Когнитивный Экзоскелет для LLM

Набор алгоритмов и философских фильтров, превращающий LLM из вежливого болтуна в хладнокровного инженера-архитектора.

**Проблема:** 90% ответов LLM — убедительный, отформатированный мусор. Модель угадывает "Best Practice" из обучающей выборки вместо того, чтобы думать.

**Решение:** ABC — когнитивный конвейер, который принудительно прогоняет задачу через 8 философских фильтров, 13 фреймворков и 8 инженерных осей выживания *до* генерации первого слова ответа.

## Quick Start

### Самый быстрый способ (любой web-чат)
1. Скачайте один файл: [`03_АВТОНОМНЫЙ_ПАЙПЛАЙН.md`](docs/02_ИНСТРУМЕНТЫ/03_АВТОНОМНЫЙ_ПАЙПЛАЙН.md)
2. Прикрепите его в чат ChatGPT / Claude / Gemini
3. Напишите задачу — модель выполнит полный цикл анализа

Этот файл — self-contained версия всего конвейера. Один файл, ноль настройки.

### Полная установка (AI-IDE)

Клонируйте репо. Дальше зависит от среды:

| IDE | Что делать |
|---|---|
| **Zed** | Ничего. `.rules` подхватывается автоматически. Пишите `abc [задача]` в чат |
| **Cursor** | Cursor не читает `.rules` — создайте симлинк или копию: `cp .rules .cursorrules` |
| **Claude Code** | Ничего. Работает через `CLAUDE.md` (уже в репо) |

## До / После

**Задача:** *"У нас микросервисы тормозят, что делать?"*

**Без ABC** (типичный ответ LLM):
> "Рекомендую оптимизировать запросы к базе данных, внедрить кеширование Redis, рассмотреть переход на gRPC, использовать connection pooling..."

Шаблонный список без диагностики. Не известно, что именно тормозит.

**С ABC** (после прогона через конвейер):
> **Задача отклонена (Фаза 0).** Отсутствуют метрики: p99 latency, RPS, flamegraph, трейсы. "Тормозят" — это языковая иллюзия (Фильтр Витгенштейна). Предоставьте: 1) трейс конкретного медленного запроса, 2) график latency за последнюю неделю, 3) текущий CPU/RAM на узлах.

ABC заставил модель *отказаться решать несуществующую проблему* и потребовать физические данные.

## Ключевые механизмы

**8 Абсолютных Фильтров** — пре-чеклист по 8 независимым измерениям невалидности задачи:
1. Витгенштейн (Язык) — реальный конфликт или языковая иллюзия?
2. Декарт (Знание) — в чём я математически уверен?
3. Сенека (Действие) — это в зоне контроля?
4. Аристотель (Цель) — какова истинная цель (Телос) системы?
5. Джеймс (Результат) — решение изменит физическую реальность завтра?
6. Хаммурапи (Причинность) — какой базовый контракт нарушен?
7. Сунь Цзы (Время) — сейчас ли момент действовать?
8. Платон (Масштаб) — я смотрю на правильный уровень?

**Инженерный Октагон** — 8 осей, по которым проверяется любое решение:
Топология → Метаболизм → Кинематика → Наследственность → Иммунитет → Гомеостаз → Мутагенез → Телос

**Двойная Спираль** — каждая задача обязана двигать Product (решение) и Process (улучшение системы) одновременно. Одноразовые фиксы запрещены.

**Кросс-доменные инварианты** — фундаментальные законы из 6 областей (термодинамика, кибернетика, биология, теория игр, когнитивистика, семиотика), заземлённые в инженерные паттерны.
