# ABC: Когнитивный Экзоскелет для LLM

Набор алгоритмов и философских фильтров, превращающий LLM из вежливого болтуна в хладнокровного инженера-архитектора.

**Проблема:** 90% ответов LLM — убедительный, отформатированный мусор. Модель угадывает "Best Practice" из обучающей выборки вместо того, чтобы думать.

**Решение:** ABC — когнитивный конвейер, который принудительно прогоняет задачу через 6 философских фильтров, 13 фреймворков и 8 инженерных осей выживания *до* генерации первого слова ответа.

## Quick Start (30 секунд)

### Вариант A: Web-чат (ChatGPT / Claude / Gemini)
1. Откройте чат с любой LLM
2. Прикрепите файл [`docs/02_ИНСТРУМЕНТЫ/03_АВТОНОМНЫЙ_ПАЙПЛАЙН.md`](docs/02_ИНСТРУМЕНТЫ/03_АВТОНОМНЫЙ_ПАЙПЛАЙН.md)
3. Напишите свою задачу. Модель выполнит полный цикл анализа

### Вариант B: AI-IDE (Zed / Cursor / Claude Code)
- **Zed:** `.rules` подхватывается автоматически. Пишите `abc [задача]` в чат
- **Cursor:** скопируйте `.rules` → `.cursorrules`
- **Claude Code:** работает через `CLAUDE.md` (уже в репо)

## До / После

**Задача:** *"У нас микросервисы тормозят, что делать?"*

**Без ABC** (типичный ответ LLM):
> "Рекомендую оптимизировать запросы к базе данных, внедрить кеширование Redis, рассмотреть переход на gRPC, использовать connection pooling..."

Шаблонный список "Best Practices" без диагностики. Не известно, что именно тормозит.

**С ABC** (после прогона через конвейер):
> **Задача отклонена (Фаза 0).** Отсутствуют метрики: p99 latency, RPS, flamegraph, трейсы. "Тормозят" — это языковая иллюзия (Фильтр Витгенштейна). Предоставьте: 1) трейс конкретного медленного запроса, 2) график latency за последнюю неделю, 3) текущий CPU/RAM на узлах.

ABC заставил модель *отказаться решать несуществующую проблему* и потребовать физические данные.

## Ключевые механизмы

**6 Абсолютных Фильтров** — философский пре-чеклист. Если задача не проходит хотя бы один — она аннулируется как когнитивное искажение:
1. Витгенштейн — это реальный конфликт или языковая иллюзия?
2. Декарт — в чём я математически уверен?
3. Сенека — это в моей зоне контроля?
4. Аристотель — какова истинная цель (Телос) системы?
5. Джеймс — решение изменит физическую реальность завтра?
6. Шумеры — какой базовый контракт нарушен?

**Инженерный Октагон** — 8 осей, по которым проверяется любое решение:
Топология → Метаболизм → Кинематика → Наследственность → Иммунитет → Гомеостаз → Мутагенез → Телос

**Двойная Спираль** — каждая задача обязана двигать Product (решение) и Process (улучшение системы) одновременно. Одноразовые фиксы запрещены.

**Кросс-доменные инварианты** — библиотека фундаментальных законов из 6 областей (термодинамика, кибернетика, биология, теория игр, когнитивистика, семиотика), заземлённых в инженерные паттерны.

## Для кого это
